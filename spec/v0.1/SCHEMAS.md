# Intent Protocol — JSON Schemas

## 1. Base Message Envelope

Every message conforms to this envelope:

```json
{
  "$schema": "https://intentprotocol.org/schemas/v0.1/envelope.json",
  "type": "object",
  "required": ["proto", "type", "id", "from", "ts", "sig"],
  "properties": {
    "proto":  { "type": "string", "const": "intent/0.1" },
    "type":   { "type": "string", "enum": ["rfq", "bid", "accept", "deal", "cancel", "receipt"] },
    "id":     { "type": "string", "description": "ULID or UUIDv7" },
    "ref":    { "type": ["string", "null"], "description": "Parent message ID" },
    "from":   { "type": "string", "pattern": "^agent:.+@.+$" },
    "to":     { "type": ["string", "null"], "description": "Target agent (null = broadcast)" },
    "ts":     { "type": "integer", "description": "Unix timestamp (seconds)" },
    "ttl":    { "type": "integer", "minimum": 0, "default": 30 },
    "sig":    { "type": "string", "description": "Ed25519 signature (base64)" },
    "via":    { "type": "array", "items": { "type": "string" }, "description": "Relay chain (federation)" }
  }
}
```

## 2. RFQ (Request For Quote)

```json
{
  "proto": "intent/0.1",
  "type": "rfq",
  "id": "01JQXYZ123ABC",
  "ref": null,
  "from": "agent:jarvis@relay.openclaw.ai",
  "ts": 1741276800,
  "ttl": 30,
  "sig": "ed25519:...",

  "intent": {
    "action": "book",
    "category": "services.beauty.haircut",

    "when": {
      "after": "2026-03-06T13:00:00Z",
      "before": "2026-03-06T17:00:00Z",
      "duration_min": 30,
      "prefer": "earliest"
    },

    "where": {
      "lat": 43.295,
      "lon": -0.370,
      "radius_km": 3,
      "mode": "provider_location"
    },

    "budget": {
      "max": 30,
      "currency": "EUR",
      "prefer": "cheapest"
    },

    "specs": {
      "service": "coupe homme",
      "language": "fr"
    },

    "quantity": 1,
    "flexible": false
  }
}
```

### Intent Actions

| Action | Description | Example |
|--------|-------------|---------|
| `book` | Reserve a time slot | Haircut, restaurant, hotel |
| `buy` | Purchase a good or service | Product, one-time service |
| `rent` | Temporary usage | Car, equipment, space |
| `hire` | Engage for a task | Freelance, repair, delivery |
| `quote` | Get a price estimate only | Insurance, construction |
| `info` | Request information | Opening hours, availability |

## 4. ACCEPT

```json
{
  "proto": "intent/0.1",
  "type": "accept",
  "id": "01JQXYZ789GHI",
  "ref": "01JQXYZ456DEF",
  "from": "agent:jarvis@relay.openclaw.ai",
  "to": "agent:salon-bella@relay.pau.fr",
  "ts": 1741276802,
  "ttl": 10,
  "sig": "ed25519:...",

  "accepted_bid": "01JQXYZ456DEF",
  "settlement": {
    "method": "direct",
    "pay_at": "on_site"
  }
}
```

### Settlement Methods

| Method | Description |
|--------|-------------|
| `direct` | Pay directly (on site, bank transfer) — no escrow |
| `escrow_stripe` | Funds held via Stripe Connect |
| `escrow_crypto` | Funds locked in smart contract (chain specified) |
| `escrow_relay` | Relay acts as escrow (for relays that offer this) |
| `invoice` | Business sends invoice after service |

  "type": "deal",

      "agent": "agent:jarvis@relay.openclaw.ai",

## 7. RECEIPT (Fulfillment Confirmation)

```json
{
  "proto": "intent/0.1",
  "type": "receipt",
  "id": "01JQXYZRECEIPT",
  "ref": "01JQXYZ999JKL",
  "from": "agent:salon-bella@relay.pau.fr",
  "ts": 1741281600,
  "sig": "ed25519:...",

  "fulfillment": {
    "completed": true,
    "actual_price": 28.00,
    "actual_duration_min": 25,
    "rating_client": 5,
    "note": null
  }
}
```

Both parties send a receipt. When both are received, the deal state transitions to `FULFILLED`.

## 8. Category Schema Registry

Categories can have associated schemas defining expected `specs` fields:

```json
{
  "category": "services.beauty.haircut",
  "version": "1.0",
  "specs_schema": {
    "type": "object",
    "properties": {
      "service": {
        "type": "string",
        "enum": ["coupe homme", "coupe femme", "coupe enfant", "coloration", "mèches", "brushing"]
      },
      "hair_length": {
        "type": "string",
        "enum": ["short", "medium", "long"]
      },
      "extras": {
        "type": "array",
        "items": { "type": "string", "enum": ["barbe", "shampoing", "soin", "coiffage"] }
      },
      "language": { "type": "string" }
    }
  }
}
```

Category schemas are **MANDATORY for defined categories** (see SECURITY.md §5.1.A). The relay MUST validate `specs` against the category schema and reject non-conforming messages with `E_INVALID`.

**Security constraints on all `specs` fields:**
- String values: max 200 characters
- No control characters (`\n`, `\r`, `\t` forbidden in values)
- Prefer `enum` types over free-text
- Free-text `note` field: max 100 chars, alphanumeric + basic punctuation only (regex: `/^[a-zA-Z0-9àâäéèêëïîôùûüÿçÀÂÄÉÈÊËÏÎÔÙÛÜŸÇ\s.,!?\-]{0,100}$/`)
- Agents MUST NOT pass specs fields as raw text into LLM prompts (see SECURITY.md §5.1.C, RULE-INJECT-01)

For categories without a defined schema, the relay applies generic validation (size limits, charset restrictions) but does not validate field names or values.
